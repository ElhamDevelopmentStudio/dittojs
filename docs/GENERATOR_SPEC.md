# DittoJs Generator Specification

## Purpose

The generator turns a resolved DittoJs recipe into a real project folder.

The generator must never decide dependency logic itself. It receives a `ResolvedRecipe` from the resolver and writes the corresponding files.

## Core rule

The generator does not resolve dependencies.

The generator only writes output from a valid resolved recipe.

## Inputs

```ts
type GenerateInput = {
  resolvedRecipe: ResolvedRecipe
  outputDir: string
  options?: GenerateOptions
}
````

## Outputs

```ts
type GenerateResult = {
  outputDir: string
  filesWritten: string[]
  packageJsonPath: string
  metadataPath: string
  warnings: string[]
}
```

## CLI contract

Early MVP should expose a CLI command:

```bash
ditto generate --preset react-recommended --out ./fixtures/generated/react-recommended
```

And:

```bash
ditto generate --recipe ./fixtures/recipes/chat-app.json --out ./fixtures/generated/chat-app
```

## Generator responsibilities

The generator must:

* Create the output directory.
* Write source files.
* Write configuration files.
* Write `package.json`.
* Write README.
* Write Ditto metadata.
* Copy or render template files.
* Resolve slot file mappings through the selected project structure adapter.
* Prevent unsafe paths.
* Report file write results.
* Fail clearly on invalid input.

## Generator non-responsibilities

The generator must not:

* Decide what dependencies are required.
* Decide what modules are selected.
* Decide conflicts.
* Mutate user selections.
* Contain UI logic.
* Call AI.
* Hardcode one-off template combinations.

## Output project structures for MVP

A generated React/Vite template uses one selected project structure. React MVP structures are:

```txt
structure.react.simple
structure.react.feature-based
structure.react.route-colocated
```

The selected structure is resolved from `ResolvedRecipe.effectiveSelections`. Static root files can use direct mappings. Source files that vary by layout should use structure slots.

Simple output looks like:

```txt
generated-project/
  package.json
  tsconfig.json
  vite.config.ts
  index.html
  README.md
  ditto.generated.json

  src/
    main.tsx
    App.tsx
    index.css

    components/
      ui/
        button.tsx
        input.tsx
        textarea.tsx
        label.tsx
        avatar.tsx
        dropdown.tsx
        sheet.tsx
        form.tsx

      blocks/
        navbar.tsx
        sidebar.tsx
        messaging-input.tsx
        typing-indicator.tsx
        online-presence.tsx

    lib/
      utils.ts
      axios.ts

    stores/
      app-store.ts
```

Only selected files should be generated.

Feature-based output places feature-owned blocks and forms under `src/features/{feature}/...`.

Route-colocated output places route-owned forms, pages, and schemas under `src/pages/{route}/...`.

## Metadata file

Every generated project must include:

```txt
ditto.generated.json
```

Example:

```json
{
  "generator": "dittojs",
  "generatorVersion": "0.1.0",
  "createdAt": "2026-07-05T00:00:00.000Z",
  "preset": "preset.react-recommended",
  "projectStructure": "structure.react.simple",
  "userSelections": [
    "preset.react-recommended"
  ],
  "effectiveSelections": [
    "framework.react",
    "styling.tailwind",
    "ui.shadcn",
    "form.react-hook-form",
    "validation.zod"
  ]
}
```

## Package generation

The generator should merge package data collected by the resolver.

Example output:

```json
{
  "scripts": {
    "dev": "vite",
    "build": "tsc -b && vite build",
    "preview": "vite preview",
    "typecheck": "tsc --noEmit"
  },
  "dependencies": {},
  "devDependencies": {}
}
```

## File rendering

Template files may be static or rendered.

Examples:

```txt
templates/react/vite.config.ts
templates/react/src/main.tsx
templates/components/ui/button.tsx
templates/blocks/navbar.tsx
```

The generator should support variable interpolation later, but MVP can start with static templates.

Slot mappings must be resolved before files are copied. A mapping with both `to` and `slot`, or with neither target, must fail clearly.

## Path safety

The generator must reject dangerous paths.

Invalid output target examples:

```txt
../outside-project/file.ts
/absolute/path/file.ts
src/../../bad.ts
```

The generator must guarantee that every written file stays inside `outputDir`.

## File collisions

The generator must fail if two selected modules write to the same target path unless the collision is explicitly allowed.

Example:

```txt
component.form.rhf-zod writes src/components/ui/form.tsx
component.form.tanstack-zod writes src/components/ui/form.tsx
```

Both should not be selected together.

## README generation

Generated projects should include a README containing:

* Project name.
* Generated by DittoJs.
* Stack summary.
* Included components.
* Included blocks.
* Install command.
* Dev command.
* Build command.
* Regeneration metadata note.

## Generated template validation

Generated templates must be able to:

```bash
pnpm install
pnpm typecheck
pnpm build
```

MVP may use pnpm as the first package manager.

## Fixture generation

The generator should support generating fixtures for CI.

Fixtures should live in:

```txt
fixtures/generated/
```

Recipes should live in:

```txt
fixtures/recipes/
```

## Required tests

Generator tests must cover:

* writing files
* package.json generation
* metadata generation
* safe path validation
* file collision handling
* project structure slot resolution
* preset generation
* recipe generation
* minimal template generation
* maximal template generation

## Golden rule

If the generated project does not install, typecheck, and build, the generator is not done.
