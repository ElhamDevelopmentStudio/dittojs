# Support

## Usage and design questions

Use [GitHub Discussions](https://github.com/ElhamDevelopmentStudio/dittojs/discussions) for setup help, manifest design questions, proposed integrations, and general project conversation. Search existing discussions and documentation first.

## Bugs

Use the [bug report form](https://github.com/ElhamDevelopmentStudio/dittojs/issues/new?template=bug.yml) for reproducible defects in supported code. Include the smallest reproduction, environment details, and relevant logs.

The issue tracker is not a guaranteed support channel or service-level agreement. Maintainers prioritize security, regressions, data-loss risks, and broadly reproducible problems.

## Feature requests

Use the [feature request form](https://github.com/ElhamDevelopmentStudio/dittojs/issues/new?template=feature.yml). Explain the user problem before proposing an implementation.

## Security

Never report vulnerabilities publicly. Follow [SECURITY.md](SECURITY.md).
