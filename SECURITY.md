# Security Policy

## Supported versions

DittoJs is currently pre-1.0. Security fixes are applied to the latest code on the default branch and, after releases begin, to the latest published minor line when practical. Older snapshots and generated projects are not automatically updated by this repository.

| Version            | Supported   |
| ------------------ | ----------- |
| Default branch     | Yes         |
| Latest pre-release | Best effort |
| Older snapshots    | No          |

## Reporting a vulnerability

Do not disclose suspected vulnerabilities in public issues, discussions, pull requests, or social media.

Use GitHub's **Report a vulnerability** button on the repository Security tab to open a private security advisory:

https://github.com/ElhamDevelopmentStudio/dittojs/security/advisories/new

Include, when available:

- Affected component and revision.
- Reproduction steps or a minimal proof of concept.
- Expected and observed impact.
- Whether generated archives or generated projects are affected.
- Suggested mitigations.
- Any disclosure deadline you are working under.

The maintainer will acknowledge a complete report as soon as practical, normally within five business days. Updates will be provided through the private advisory. Timelines depend on severity, reproducibility, and release impact.

## Disclosure process

1. Confirm and scope the report privately.
2. Prepare a fix and regression coverage.
3. Coordinate an advisory and release when users need an update.
4. Credit the reporter unless anonymity is requested.
5. Publish remediation guidance after affected users can upgrade.

Please act in good faith, avoid privacy violations and service disruption, and give maintainers a reasonable opportunity to remediate before public disclosure.
