# Roadmap

This roadmap communicates direction, not a delivery promise. Accepted issues and release milestones remain the authoritative planning surfaces.

## Now: reliable MVP

- Stabilize the React + Vite + TypeScript generation path.
- Keep resolver requirements, locks, and conflicts fully explainable.
- Expand generated-project tests and fixture coverage.
- Improve builder accessibility, responsive usability, and recovery states.
- Document manifest authoring and registry contribution workflows.
- Establish secure, reproducible GitHub releases.

## Next: public extension model

- Define stable public APIs for core, catalog, and generator packages.
- Introduce a supported CLI generation workflow.
- Formalize catalog and registry validation tooling.
- Add controlled package/version upgrade workflows.
- Support reusable user recipes and import/export.
- Publish contributor-oriented examples and extension templates.

## Later: broader ecosystem

- Additional framework and UI-system adapters.
- Saved templates and regeneration history.
- GitHub export.
- Team and workspace presets.
- Public and private registries.
- AI-assisted recommendations that produce schema-valid recipes and always pass through the resolver.

## Explicit non-goals

DittoJs is not becoming a no-code page builder, a runtime framework, or an unconstrained AI code generator. Generated projects must remain independent, understandable, and owned by their users.
